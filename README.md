# signals — анализатор АЧХ

Приложение для измерения и анализа амплитудно-частотной характеристики
резонансных систем. Плагинная архитектура: приборы, колонки-метрики, функции
выражений и парсеры добавляются как плагины, без правок ядра.

Статус по этапам — см. `STAGES.md`. Сейчас готовы этап 1 (рефакторинг) и
этап 2 (полное восстановление функционала, GUI).

## Установка

```bash
python -m venv .venv
# Windows:  .venv\Scripts\activate     Linux/macOS:  source .venv/bin/activate
pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

## Быстрая проверка без прибора

```bash
python tools/make_sample.py ./sample      # создаст два образцовых CSV
python main.py
```
Далее в окне: **Добавить измерения** → выбрать оба CSV из `sample/`
(имена `Деталь-A_1000_100000_1.csv` распознаются: код объекта, старт/кон. частота,
время записи подставятся автоматически). Кнопка **«Открыть графики»** в строке — окно с
подстройкой параметров. Отметь галочками два измерения → **Сводка** → введи
выражение `g1 / g2` → **Вычислить** → **Экспорт в Excel**. **Сохранить** —
проект сложится в папку (инкрементально), **Экспорт .sigproj** — в один файл.

## Возможности (этап 2 — интерфейс приведён к оригиналу)

* **Стиль виджетов: Fusion по умолчанию** (а не «отвратительный» нативный
  Windows 11). Меню **«Вид» → «Стиль интерфейса»**: Fusion, **Windows 10**,
  Windows 11, классический — на выбор. Плюс **темы** (тёмная/светлая/«Системная
  без темы»/свой `.qss`). И стиль, и тема запоминаются.
* **Дерево «Предмет → измерение»** со столбцами как в оригинале: Выбор · Код
  предмета · Файл/измерение · **«Открыть графики» (кнопка в каждой строке)** ·
  динамические столбцы.
* **Имя предмета и метку измерения** меняешь прямо в ячейке (двойной клик); у
  измерения есть **метка и описание** (описание — в окне графиков, видно подсказкой).
* **Фильтр — свойство столбца**: двойной клик по заголовку открывает фильтр этого
  столбца (Все/Равно/Больше/Меньше/Между; для предмета — текстовый); «Отключить фильтры».
* **Настройки столбцов** — диалог «Доступные ↔ Текущие» (добавить/убрать/вверх/вниз/
  по умолчанию). Значения берутся **из процессора**: резонанс, полоса −3 дБ, полоса
  по фикс. уровню, добротность, макс. амплитуда, сырые макс/мин, время начала
  анализа, обрезка/усиление/уровень. Плюс свой столбец-формула.
* **Окно графиков**: три графика одновременно (сигналы со «стробом», сглаженные,
  АЧХ с маркерами резонанса/0.707/уровня), живой предпросмотр, «Применить»/«Закрыть»,
  справа — параметры канала и прогноз полосы.
* **Приборы — три отдельные группы**: «Генератор сигналов» (частоты, амплитуда,
  смещение, **время развёртки** — свип по времени, без «точек»), «Осциллограф»
  (чтение + периодическая запись), «Управление». **Цветные кнопки** «НАЧАТЬ ЗАПИСЬ»
  (синяя) / «ОСТАНОВИТЬ» (красная) / периодическая запись (зелёная↔красная) с
  **динамическими состояниями**. Значения по умолчанию — подобранные.
* **Сохранить всё / сохранить выбранные**; **загрузка старого формата `.analysis`**;
  перенос анализов между предметами (drag-drop и контекстное меню); удаление по
  **Delete**; запрос сохранения при выходе; автосохранение; `.sigproj`.
* Сводка: наложение АЧХ, выражения над датасетами, экспорт в Excel.

## Прибор Hantek DSO-6104BD (этап 3 — поддержка DLL)

Прибор управляется через `HTHardDll.dll` (USB, `__stdcall`). Чтобы приложение его
нашло, положите DLL Hantek, **разделив по разрядности Python**:

* 64-битный Python → `signals/contrib/hantek_dll/x64/`
* 32-битный Python → `signals/contrib/hantek_dll/x86/`

Кладите все DLL из SDK (`HTHardDll.dll`, `HTSoftDll.dll`, `HTDisplayDll.dll`,
`HTMeasDll.dll` и зависимости). Можно также указать папку через меню
**«Приборы → Указать папку с DLL Hantek…»** или переменную `HANTEK_DLL_DIR`.

**Важно про разрядность.** DLL и Python должны совпадать. SDK Hantek часто
32-битный — тогда либо возьмите 64-битную DLL, либо запустите 32-битный Python.
При несовпадении/отсутствии DLL в **журнале** будет понятное сообщение, а после
размещения DLL «Обновить список приборов» покажет, какая DLL загружена и какие
приборы откликнулись (удобно для снятия фингерпринтов).

Боевая проверка чтения/свипа на реальном приборе — на машине с прибором; в коде
отмечены `TODO(device)` (упаковка структур и частота на быстрых развёртках).

## Архитектура (без Qt — ядро; Qt только в app_qt)

```
signals/
  plugins/      реестр + автообнаружение + capabilities
  domain.py     Channel / Analysis / Subject / Project
  engine.py     расчёт АЧХ и временных рядов
  io/           парсеры + хранилище проекта (.sigproj)
  services/     analyze_full, MeasurementService (свип)
  instruments.py + contrib/inst_*   приборы (Hantek/Tektronix/Rigol)
  contrib/      встроенные плагины (метрики, функции, стратегии фронта)
  app_qt/       GUI (главное окно, графики, сводка, панель прибора)
main.py         точка входа
tools/make_sample.py   генератор тестовых данных
```
