# signals — анализатор АЧХ

Программа для измерения и анализа амплитудно-частотной характеристики
резонансных систем (например, калибровки керамических антенн). Архитектура
плагинная: новый прибор, столбец-метрика, функция выражений или формат файла
добавляются отдельным модулем, без правок ядра — см. `docs/plugins.md`.

Что сделано и в какой последовательности — `STAGES.md`.

## Установка

```bash
python -m venv .venv
# Windows:  .venv\Scripts\activate     Linux/macOS:  source .venv/bin/activate
pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

## Прибор Hantek DSO-6104BD

Управляется через `HTHardDll.dll` (USB, вызовы `__stdcall`). Чтобы программа его
нашла, нужно положить DLL из комплекта Hantek в папку, соответствующую разрядности
Python:

* 64-битный Python → `signals/contrib/hantek_dll/x64/`
* 32-битный Python → `signals/contrib/hantek_dll/x86/`

Кладите весь набор из SDK: `HTHardDll.dll`, `HTSoftDll.dll`, `HTDisplayDll.dll`,
`HTMeasDll.dll` и их зависимости. Папку с DLL можно также указать через меню
«Приборы → Указать папку с DLL Hantek…» или переменной окружения `HANTEK_DLL_DIR`.

Разрядность DLL и Python должны совпадать — SDK Hantek чаще всего 32-битный,
так что либо ищите 64-битную сборку DLL, либо запускайте программу под 32-битным
Python. Если разрядности не совпали или DLL не нашлась — в журнале появится
понятное сообщение; а когда DLL загрузилась, «Обновить список приборов» покажет,
какая именно DLL подхватилась и что откликнулось на сканирование (полезно, если
приборов несколько и нужно понять, какой из них какой).

Драйвер прогнан на реальном DSO-6104BD — подключение, чтение каналов и
свип-захват работают; подробности и единственная оставшаяся непроверенной
вещь (быстрые развёртки) — в `signals/contrib/inst_hantek.py`.

## Архитектура (без Qt — ядро; Qt только в app_qt)

```
signals/
  plugins/      реестр + автообнаружение + capabilities
  domain.py     Channel / Analysis / Subject / Project
  engine.py     расчёт АЧХ и временных рядов
  io/           парсеры + хранилище проекта (.sigproj)
  services/     analyze_full, MeasurementService (свип)
  instruments.py + contrib/inst_*   приборы (Hantek/Tektronix/Rigol)
  contrib/      встроенные плагины (метрики, функции, стратегии фронта)
  app_qt/       GUI (главное окно, графики, сводка, панель прибора)
main.py         точка входа
tools/make_sample.py   генератор тестовых данных
```
