# signals — план по этапам

Двигаемся по этапам, чтобы на каждом шаге было что-то рабочее и проверяемое —
без затяжных переписываний, после которых непонятно, всё ли ещё работает.

| Этап | Что делаем | Статус | Где смотреть |
|------|------------|--------|--------------|
| 1 | Рефакторинг: новая архитектура, ядро без Qt | готово | `signals/plugins`, `domain`, `engine`, `io`, `services`, `instruments`, `contrib` |
| 2 | Восстановление функционала старого GUI | готово | `signals/app_qt` |
| 3 | Поддержка Hantek (DLL отдельно для x86/x64) | готово | `signals/contrib/inst_hantek.py`, `hantek_dll/{x86,x64}` |
| 4 | Автообновление с GitHub | готово | `signals/services/updater.py`, `app_qt/update_dialog.py` |
| 5 | Шаблоны обработки и формул | готово | `signals/services/templates.py`, `app_qt/templates_dialog.py` |
| 6 | Набор кривых: обмен с Excel, формулы, undo/redo | готово | `signals/services/clipboard_data.py`, `app_qt/curve_board.py` |
| 7 | Документация для тех, кто будет дописывать плагины | готово | `docs/plugins.md`, `docs/scenarios.md` |
| 8 | Обучение при первом запуске | готово | `signals/app_qt/onboarding.py`, `signals/contrib/verdict.py` |
| 9 | Правки по результатам работы на реальном железе | в работе | по мере поступления отзывов |

## Этап 1 — рефакторинг

Главная цель — выдернуть ядро из Qt, чтобы его можно было гонять и тестировать
без GUI (`python -m examples.demo` поднимает всё: автообнаружение, проверку
возможностей прибора, рантайм-колонку, расчёт АЧХ).

Как разложено:

* **Плагины** (`signals/plugins`, `extpoints.py`) — все точки расширения сведены
  в реестры: приборы, столбцы, функции выражений, стратегии поиска фронта,
  парсеры, экспортёры. Подключаются тремя путями: модуль в `contrib/`, drop-in
  файл в папке плагинов, pip-пакет с entry point. Прибор сам говорит, что умеет
  (`capabilities()`), а не определяется по списку моделей — так не приходится
  трогать GUI каждый раз, когда появляется новый прибор.
* **Доменная модель** (`domain.py`) — `Channel`, `MeasurementParams`, `Analysis`,
  `Subject`, `Project` вместо словарей с произвольными ключами. `Analysis.dirty`
  и `Project.dirty_analyses()` — на этом держится инкрементальное сохранение
  (пишем на диск только то, что реально поменялось).
* **Движок** (`engine.py`) — расчёт АЧХ и временных рядов (сырой/сглаженный/
  обрезанный сигнал), стратегия поиска фронта подставляется из реестра.
* **Ввод-вывод** (`io/`) — парсеры старых форматов (CSV, Excel), чтобы открывались
  данные прежних версий, и хранилище проекта: `project.json` + `data/<id>.csv`,
  атомарная запись (через временный файл и `os.replace`, чтобы не убить проект
  при сбое посередине), упаковка в `.sigproj` (zip) для переноса. От pickle
  отказались — слишком хрупкий формат для долгоживущих данных.
* **Сервисы** — `analyze_full` (расчёт + все колонки) и `MeasurementService`
  (свип с захватом для приборов вроде Hantek, у которых генератор и осциллограф
  совмещены).
* **Приборы** (`instruments.py`, `contrib/inst_*`) — протоколы `Oscilloscope` и
  `Generator`; Hantek собран через ctypes-биндинг по заголовкам SDK (со встроенным
  DDS-генератором), плюс заготовки под Tektronix и Rigol.

На этом этапе сознательно нет интерфейса (он в этапе 2) и нет проверки Hantek на
живом приборе — биндинг написан по документации SDK, сверку с реальным железом
делали на этапе 3.

## Этап 2 — интерфейс

Переносили старый функционал на новую архитектуру: дерево «предмет → измерения»,
графики и подстройка анализа, сводка по нескольким измерениям, сохранение/
загрузка проекта, экспорт в Excel, фильтры и сортировка по столбцам, панель
прибора с поиском/измерением/чтением. Сводный диалог и панель приборов в части
тонких возможностей (алгебра датасетов, выбор пары генератор+осциллограф)
донастраивали уже по ходу этапов 5–6, когда стало понятнее, как ими пользуются
на практике.

## Этап 7 — документация для тех, кто будет писать плагины

Две вещи:

* **`docs/plugins.md`** — справочник по архитектуре расширений: какие есть
  реестры, как устроены `Registry`/`Entry`, как подключаются плагины (встроенные,
  drop-in, pip entry point), что такое `capabilities`, какой контракт у прибора
  (протоколы `Oscilloscope`/`Generator`, `matches_idn`/`USES_VISA` для VISA-
  приборов и `discover()` для приборов без VISA), как добавлять столбцы, функции
  выражений, стратегии поиска фронта, парсеры.
* **`docs/scenarios.md`** — рецепты «как добавить X» с шаблонами кода: новый
  осциллограф, генератор, столбец (с кодом и без), функция выражений, стратегия
  фронта как drop-in, парсер формата, упаковка в pip-пакет, проверка, что плагин
  вообще подхватился.

Шаблоны из рецептов гонял через `discover()` как временные drop-in плагины —
работают без правок ядра, что и было целью всей этой реструктуризации.

## Этап 8 — обучение при первом запуске

`signals/app_qt/onboarding.py` — оверлей-прожектор: затемняет окно и подсвечивает
нужный виджет, ведёт по шагам подсказками с кнопками Назад/Далее/Пропустить и
картинками/видео образцов (антенна, вилка, форма сигнала — лежат в
`~/.signals/onboarding` или встроенной `onboarding_media`). На шагах с действием
(нажать кнопку, развернуть предмет) подсветка не блокирует приложение — реальный
виджет остаётся кликабельным, и переход к следующему шагу происходит сам, когда
действие выполнено.

Два тура (доступны через меню «Справка» и при первом запуске через
`WelcomeDialog`):

* **Базовый — калибровка антенн на вилке К-3.** Прибор → вилка → антенна с
  моментом затяжки 3.5 → переходная плата по ключам → находим прибор → задаём
  диапазон 2155–2185 Гц, амплитуду 1 В, развёртку 30 с (тур сам подставляет эти
  значения в подсвеченные поля) → добавляем предмет → проверяем, что вилка висит
  свободно → запускаем запись → смотрим график, сверяем форму с образцом →
  получаем вывод о годности по амплитуде (для этого добавляется столбец «Годность
  антенны», порог настраивается).
* **Продвинутый — обзор возможностей.** Свип-АЧХ, простое чтение осциллографа,
  периодическая запись, дерево с фильтрами и сортировкой, обработка и графики,
  столбцы и формулы, сводка и набор кривых, шаблоны, экспорт, автообновление,
  расширение плагинами. Акцент на том, что приложение работает не только с
  калибровкой антенн — оно снимает и обрабатывает любые осциллограммы с
  поддерживаемых приборов.

Колонка-вердикт вынесена отдельным плагином — `signals/contrib/verdict.py`
(«Годность антенны» по максимальной амплитуде против порога `verdict_threshold`
из настроек).

Меню «Справка» сделано как в обычных программах: можно перезапустить любой тур
целиком или сразу перейти к нужному шагу, плюс «О программе».

## Этап 9 — правки по результатам работы на железе

Здесь копится то, что вылезает уже при реальном использовании — список будет
расти по ходу дела.

**Обучение:**
* шаг про поиск прибора подсвечивает сами списки осциллографа/генератора, а не
  кнопку «Обновить» — раньше человек смотрел не туда;
* картинка-иллюстрация сразу видна миниатюрой в подсказке, не нужна отдельная
  кнопка «показать»;
* шаг с запуском измерения теперь ждёт сигнала `captured` (конец записи), иначе
  следующий шаг пытался подсветить кнопку графиков раньше, чем она появлялась;
* подсветка переехала внутрь главного окна отдельным прозрачным для мыши слоем
  (`WA_TransparentForMouseEvents`) — раньше отдельное окно поверх всего перекрывало
  диалоги настройки измерения и конфликтовало с модальными окнами (диалог
  сохранения при выходе, фильтры, окно графиков blocked друг друга); теперь они
  просто перекрывают обучение и возвращают его при закрытии;
* шаг про следующую антенну прямо говорит: для каждой — новый предмет;
* продвинутый тур переписан конкретнее, подсвечивает реальные пункты меню
  (Шаблоны, Обновление, Кривые, Справка);
* шаг «прибор найден?» реагирует на реальное состояние: нет осциллографа —
  подсвечивает «Обновить список» и просит проверить USB/драйверы и журнал; есть —
  подтверждает и предлагает выбрать нужный из списка; не предлагает жать
  «Обновить», пока поиск уже идёт и кнопка неактивна;
* добавлена защита от повторного запуска тура (старый оверлей закрывается перед
  новым).

**По мелочи:**
* подсветка заголовков дерева больше не захватывает все столбцы при единственном
  предмете (`highlightSections=False`) — путала с сортировкой по клику;
* к именам измерений добавляется временная метка `ЧЧ:ММ:СС_ДД/ММ/ГГГГ`;
* адаптивный поиск фронта стал устойчивее на записях без участка тишины — базовый
  уровень теперь берётся по нижнему перцентилю всей записи, а не первого отрезка;
* предзапись увеличена до 2 секунд — участок без излучения стал заметнее на
  графике.
