## 1. Окно целиком

Вертикально:
1. Заголовок «Структура предметов и анализов».
2. **Горизонтальная панель кнопок** (в одну строку, с вертикальными разделителями):
   `Добавить предмет` | `Добавить измерения` | `Загрузить анализ` ‖ `Сохранить всё` |
   `Сохранить выбранные` ‖ `Настройки столбцов` | `Отключить фильтры` ‖ `📈 Сводка`.
3. **Дерево** предметов/анализов (центр).
4. **Секция приборов** (внизу) — отдельный большой блок.

Delete — удаляет выбранный анализ/предмет. При выходе — вопрос «Сохранить данные перед выходом?» (Да/Нет/Отмена).

## 2. Дерево: столбцы и поведение

Колонки (фиксированный левый блок + динамические справа):
`[0] Выбор (чекбокс)` · `[1] Код предмета` · `[2] Файл анализа` ·
`[3] Графики и подстройка значений (КНОПКА «Открыть графики» в каждой строке)` ·
`[4…] динамические столбцы`.

* **Предмет** (`SubjectItem`): редактируемый (`ItemIsEditable`) — **имя детали меняется прямо в ячейке** (двойной клик по коду предмета). Является целью drop.
* **Анализ** (`AnalysisItem`): чекбокс (по умолчанию включён), имя файла, кнопка «Открыть графики», динамические значения. Источник перетаскивания (drag).
* **Перетаскивание**: анализ можно перетащить в другой предмет (drag-drop), данные при этом переносятся.
* Имя файла измерения генерируется автоматически: `{имя предмета}_{старт}_{полоса}_{время записи}_{чч:мм:сс_дд/мм/гггг}.csv`. Своей «метки» и «описания» у измерения в оригинале НЕ было (это — улучшение, которое нужно добавить).

## 3. Столбцы (значения из процессора) и их настройка

«Настройки столбцов» открывают **модальный диалог** «Настройка столбцов»:
два списка — **«Доступные параметры» ↔ «Текущие столбцы»**, перетаскивание/кнопки
`Добавить →`, `← Удалить`, `Вверх`, `Вниз`, и `RestoreDefaults`.
По умолчанию показываются `start_freq, end_freq, record_time`.

**Доступные столбцы берутся из процессора** (это и есть «стандартные значения, которые уже были»):

| Ключ | Заголовок | Источник |
|------|-----------|----------|
| cut_second | Обрезка (сек) | params |
| gain | Усиление | params |
| fixedlevel | Фиксированный уровень | params |
| max_amplitude | Макс. амплитуда (В) | channel_params |
| resonance_frequency | Резонансная частота (Гц) | channel_params |
| bandwidth_707 | Полоса −3 дБ (Гц) | channel_params |
| bandwidth_fixed | Полоса фикс. уровня (Гц) | channel_params |
| q_factor | Добротность | channel_params |
| raw_max_amp | Макс. амплитуда (сырая) | raw_data |
| raw_min_amp | Мин. амплитуда (сырая) | raw_data |
| analysis_start_time | Время начала анализа | processor |

## 4. Фильтры — это СВОЙСТВО СТОЛБЦА, а не отдельная панель

`header().sectionDoubleClicked` → **двойной клик по заголовку столбца открывает диалог
фильтра именно этого столбца**: тип `Все значения / Равно / Больше / Меньше / Между`,
поле «Значение» и поле «До» (активно только для «Между»). Для столбца «Код предмета» —
текстовый фильтр «содержит». Кнопка «Отключить фильтры» сбрасывает все.

## 5. Окно графиков (кнопка «Открыть графики» в строке анализа)

Три колонки: **слева** настройки, **по центру** ТРИ графика друг под другом
(`311/312/313`), **справа** «Параметры канала» + «Прогноз полосы частот».

Слева поля: «Канал для определения начала сигнала», «Канал для анализа»,
«Стартовая частота», «Конечная частота», «Время записи», «Относительное смещение
строба», «Коэффициент усиления», «Пороговый уровень», кнопки «Применить значения»
и «Закрыть».

Графики: (1) исходные сигналы + красный прямоугольник «строб» анализируемого окна +
вертикаль начала; (2) сглаженные; (3) АЧХ выбранного канала с маркерами: резонанс
(зелёная пунктирная), уровень 0.707 (синяя), пороговый уровень (оранжевая).
**Живой предпросмотр**: смена частот/времени перерисовывает графики; усиление,
уровень, смещение и каналы применяются сразу; «Применить» фиксирует в процессор.
Навигационная панель matplotlib переведена на русский и показывает координаты курсора.

Справа «Параметры канала»: Макс. амплитуда (×2 для отображения), резонанс, полоса
0.707 + диапазон, полоса по фикс. уровню + диапазон, добротность. «Прогноз полосы»:
критерий достаточности (Гц/с) → нижняя/верхняя/центр, формула
`резонанс ± (критерий · время_записи)/2`.

## 6. Значения по умолчанию (именно эти, подобранные)

```
start_freq = 100   end_freq = 1000   amplitude = 1   offset = 0
sweep_time = 30    record_time = 1   cut_second = 0  fixedlevel = 0.6   gain = 7
```
Эти значения подставляются в поля генератора и как стартовые в окне графиков —
ничего переключать не нужно. (У меня сейчас стояли чужие: 1000/100000/…/«точек свипа».)

## 7. Приборы — ТРИ отдельные группы рядом

* **Генератор сигналов**: свой combo выбора генератора + поля «Начальная частота»,
  «Конечная частота», «Амплитуда (В)», «Смещение (В)», «Время развёртки (сек)».
* **Осциллограф**: свой combo + кнопка «Прочитать данные с осциллографа» + вложенная
  группа «Периодическая запись» (поле «Период опроса (сек)» = 5 и кнопка-переключатель).
* **Управление**: «Обновить список приборов», «НАЧАТЬ ЗАПИСЬ», «ОСТАНОВИТЬ».
* Внизу — общий прогресс-бар и лог.

**Никакого «меню точек свипа» нет** — свип задаётся ВРЕМЕНЕМ развёртки (аппаратный
линейный свип), а не числом точек. Обнаружение приборов раздельно ищет генераторы и
осциллографы, предупреждает, если нет генератора Rigol / осциллографа Tektronix.

## 8. Кнопки: цвета и ДИНАМИЧЕСКИЕ состояния

Стили из `constants.py`:
* «НАЧАТЬ ЗАПИСЬ» — синяя, белый жирный текст, рамка, hover/pressed, высота 50.
* «ОСТАНОВИТЬ» — красная, белый текст, hover.
* «Запуск периодической записи» — зелёная, при включении (`:checked`) **становится
  красной** «⏹ Остановить запись», hover-варианты.
* Палитра состояний: SUCCESS (зелёный), ERROR (красный), NORMAL (серый),
  WARNING (жёлтый), ACTIVE (синий).

Состояния меняются динамически: `set_measurement_state`, `set_reading_state`,
`set_recording_state`, `set_ui_enabled` — во время измерения/чтения/записи нужные
кнопки блокируются, кнопка записи переключает цвет. (Мой нынешний плоский стиль это не воспроизводит — переделать.)

## 9. Хранение: старый формат нужно уметь читать

Сохранение оригинала — **pickle с расширением `.analysis`** РЯДОМ с одноимённой
**папкой**, где лежат CSV. Структура пикла:
```
{
  'subjects': {
     <код>: {
        'subject_name': <имя детали>,
        'metadata': {...},
        'analyses': {
           <индекс>: {
              'file_name', 'original_file_name',
              'params': {start_freq, end_freq, record_time, cut_second, gain, fixedlevel, ...},
              'channels_data': { <канал>: {'name', 'data': DataFrame.to_dict()} }
           }
        }
     }
  },
  'files': { (<код>, <индекс>): <имя файла> }
}
```
Важно: DataFrame канала имеет колонки **«Время»** и **«Амплитуда»** (по-русски).
Есть «Сохранить выбранные» (только отмеченные) и аварийное автосохранение в `.analysis`.

→ Нужен загрузчик-миграция: распаковать pickle, восстановить каналы из
`channels_data` (колонки «Время»/«Амплитуда»), параметры и имена предметов.

## Что я нарушил и приведу к оригиналу (после твоего «ок»)

1. **Значения по умолчанию** → строго `DEFAULT_PARAMS` (100/1000/30/1/0/0.6/7), убрать «точек свипа».
2. **Приборы** → три раздельные группы: Генератор / Осциллограф / Управление; свип по времени развёртки.
3. **Фильтры** → не отдельная панель, а двойной клик по заголовку столбца (диалог фильтра столбца) + «Отключить фильтры».
4. **Столбцы** → диалог «Доступные ↔ Текущие» (перетаскивание, restore defaults), значения тянутся из процессора по таблице из §3.
5. **Имя детали** → редактирование прямо в ячейке предмета.
6. **Метка и описание измерения** → добавить как поля (улучшение поверх оригинала).
7. **Старый формат** → загрузчик `.analysis` (pickle + папка).
8. **Кнопки** → цвета и динамические состояния как в §8 (MEASURE/STOP/REC + блокировки).
9. Кнопка «Открыть графики» в каждой строке анализа (а не только двойной клик).
10. Drag-drop переноса анализов между предметами.
